# STM32 智能手表项目创新点报告

## 一、项目名称

基于 STM32F103C8T6 的智能手表设计

## 二、创新点概述

本项目围绕“低成本、可扩展、易调试”的嵌入式智能手表原型设计目标，在硬件供电、模块化外设接入、实时任务调度、菜单交互和运动数据处理等方面进行了综合设计。项目虽然采用常见的 STM32F103C8T6 最小系统板作为主控，但通过分阶段开发、模块化驱动、面包板供电验证、FreeRTOS 多任务调度和蓝牙数据通信等方式，提高了系统的完整性、可维护性和演示效果。

---

## 三、创新点一：渐进式模块化开发方案

本项目没有一开始直接将所有模块同时接入，而是采用分阶段开发方式：

1. Phase 1：OLED 显示屏点亮与字符显示；
2. Phase 2：MPU6050 姿态传感器数据读取；
3. Phase 3：HC-05 蓝牙通信；
4. Phase 4：FreeRTOS 多任务调度与 EC11 编码器交互；
5. Phase 5：计步算法、菜单框架和电源管理扩展。

这种设计方式的创新点在于：每次只新增一个硬件模块，并在上一阶段稳定运行的基础上继续开发。这样可以明显降低调试难度，避免多个模块同时出错时无法定位问题。对于课程设计项目而言，该方法比“所有模块一次性接入”的方式更稳定，也更适合多人分工协作。

---

## 四、创新点二：基于面包板的可拆卸供电验证方案

本项目供电部分没有直接焊死在洞洞板上，而是采用面包板进行模块化调试。供电链路设计为：

```text
3.7V 锂电池 → TP4056 充电保护模块 → HT7333-3.3V 稳压 → 3.3V 电源轨
```

其中，TP4056 负责锂电池充电和保护，HT7333 负责将电池电压稳定为 3.3V，为 STM32、OLED、MPU6050、EC11、HC-05 等模块供电。

该设计的创新点主要体现在：

- 供电模块可单独测试，避免直接接入 STM32 后因电压异常造成损坏；
- 面包板连接便于快速调整线路，适合课程设计阶段反复调试；
- 使用 HT7333 低压差稳压芯片，相比普通 AMS1117 更适合单节锂电池供电；
- 在接入主控前先通过万用表确认 VOUT 是否稳定在 3.25V～3.35V，提高了硬件安全性。

这种方案提高了硬件调试的容错率，也方便后续根据实物尺寸再转移到洞洞板或 PCB。

---

## 五、创新点三：OLED 与 MPU6050 共用 I2C 总线

系统中 OLED 显示屏和 MPU6050 姿态传感器均使用 I2C 通信方式。项目将二者共用 STM32 的 I2C1 总线：

```text
PB6 → OLED SCL + MPU6050 SCL
PB7 → OLED SDA + MPU6050 SDA
```

OLED 的 I2C 地址通常为 0x3C，MPU6050 的地址为 0x68，因此二者可以挂载在同一条总线上，不会产生地址冲突。

该设计的创新点在于：

- 节省 STM32 引脚资源；
- 简化硬件连接；
- 便于软件统一管理 I2C 外设；
- 为后续加入更多 I2C 模块预留扩展空间。

在 FreeRTOS 阶段，软件中通过互斥量保护 I2C 总线，避免 OLED 刷新和 MPU6050 数据读取同时访问 I2C 导致通信异常，体现了硬件总线共享与软件资源保护的结合。

---

## 六、创新点四：FreeRTOS 多任务调度架构

项目在基础裸机功能实现后，引入 FreeRTOS 实时操作系统，将原本集中在 `while(1)` 中的功能拆分为多个任务：

| 任务 | 主要功能 |
|---|---|
| 时间任务 | 实现时间计数与更新时间 |
| 传感器任务 | 周期性读取 MPU6050 数据 |
| 显示任务 | 负责 OLED 页面刷新 |
| 菜单任务 | 处理 EC11 编码器旋转和按键事件 |
| 蓝牙任务 | 负责 HC-05 数据发送和接收处理 |

该设计的创新点在于：

- 避免不同功能之间互相阻塞；
- 提高编码器按键、蓝牙通信等交互功能的响应速度；
- 使用队列传递传感器数据和菜单消息；
- 使用互斥量保护 I2C 总线；
- 使用信号量触发 OLED 刷新，减少无效刷新。

相比简单裸机循环，FreeRTOS 架构更接近实际嵌入式产品的软件组织方式，使系统具备更好的扩展性和实时性。

---

## 七、创新点五：基于 EC11 编码器的菜单交互设计

项目使用 EC11 旋转编码器作为智能手表的人机交互输入设备。编码器接线采用：

```text
EC11 A 相 → PA6 / TIM3_CH1
EC11 B 相 → PA7 / TIM3_CH2
EC11 按键 → PB10 / EXTI10
```

旋转操作用于页面切换和菜单光标移动，按键用于确认、进入菜单或返回上一级。该设计相比普通按键具有更好的交互体验。

创新点体现在：

- 通过 TIM3 编码器模式识别旋转方向和步数；
- 通过 EXTI 中断检测按键事件；
- 使用短按、长按区分不同操作；
- 建立菜单状态机，为多级菜单扩展提供基础。

虽然完整多级菜单的深层功能仍有待完善，但项目已实现菜单框架，为后续加入时间设置、蓝牙设置、步数清零、屏幕亮度调节等功能提供了接口。

---

## 八、创新点六：蓝牙串口数据通信与手机端显示

项目使用 HC-05 蓝牙模块实现 STM32 与手机之间的数据通信。HC-05 通过 USART1 与 STM32 相连：

```text
STM32 PA9  / USART1_TX → HC-05 RXD
STM32 PA10 / USART1_RX ← HC-05 TXD
```

系统可以将 MPU6050 的加速度、姿态角、温度以及计步数据通过蓝牙发送到手机端，使用手机蓝牙串口助手即可查看实时数据。

该设计的创新点在于：

- 扩展了智能手表与外部设备的数据交互能力；
- 手机端无需专门开发复杂 APP，使用串口调试工具即可验证数据；
- 为后续实现手机同步时间、远程清零步数、参数设置等功能打下基础；
- 增强了项目展示效果，使系统不仅能本地显示，还能外部通信。

---

## 九、创新点七：基于加速度峰值检测的计步算法

项目在 MPU6050 数据采集基础上实现了简易计步功能。计步算法主要依据三轴加速度合成值进行峰值检测：

```text
accel_mag = sqrt(Ax² + Ay² + Az²)
```

通过低通滤波、动态阈值和最小步间隔判断是否产生有效步伐。

该创新点主要体现在：

- 利用已有 MPU6050 模块实现运动检测，未额外增加传感器成本；
- 通过动态阈值适应不同幅度的运动变化；
- 通过最小步间隔减少误触发；
- 将姿态传感器从“数据显示”扩展到“运动分析”。

虽然计步精度仍受佩戴姿态、运动幅度和滤波参数影响，但已经实现了从原始传感器数据到运动信息提取的功能闭环。

---

## 十、创新点八：软硬件协同调试流程

项目开发过程中，硬件部分和软件部分并不是独立推进，而是通过统一的接口表和分阶段测试流程进行协同。

硬件侧完成：

- TP4056 与 HT7333 供电链路测试；
- OLED、MPU6050、HC-05、EC11 接线确认；
- 3.3V 电源轨稳定性测试；
- I2C、USART、TIM3、EXTI 对应引脚确认。

软件侧完成：

- STM32CubeIDE 工程配置；
- I2C1、USART1、TIM3、EXTI、FreeRTOS 配置；
- OLED、MPU6050、蓝牙、菜单和计步模块驱动编写；
- 功能分阶段验证。

这种软硬件协同方式使得每个问题都能定位到具体模块，例如供电问题、I2C 地址问题、串口波特率问题、编码器方向问题等，提高了整体开发效率。

---

## 十一、创新点九：低成本智能手表原型设计

本项目选用的器件均为低成本、易获取的通用模块，包括 STM32F103C8T6 最小系统板、0.96 寸 I2C OLED、MPU6050、HC-05、EC11、TP4056、HT7333 等。

该方案的特点是：

- 成本低，适合课程设计和教学实验；
- 模块化程度高，便于学生理解每个功能单元；
- 连接方式简单，适合从面包板验证逐步过渡到洞洞板或 PCB；
- 功能完整，覆盖显示、传感器、通信、交互和电源管理等嵌入式系统核心内容。

因此，本项目不仅是一个智能手表原型，也可以作为 STM32 嵌入式系统综合实践平台。

---

## 十二、创新点总结

本项目的主要创新点可以概括为：

1. 采用渐进式开发策略，降低系统调试难度；
2. 使用面包板进行供电模块验证，提高调试灵活性；
3. OLED 与 MPU6050 共用 I2C 总线，节省引脚资源；
4. 引入 FreeRTOS 多任务调度，提高系统实时性和可扩展性；
5. 使用 EC11 编码器实现菜单交互，提升用户操作体验；
6. 通过 HC-05 蓝牙实现手机端数据查看；
7. 基于 MPU6050 加速度数据实现计步算法；
8. 采用软硬件协同调试流程，提高开发效率；
9. 使用低成本模块搭建完整智能手表原型，适合课程设计展示与后续扩展。

---

## 十三、后续可扩展方向

后续可以在现有创新点基础上进一步完善：

- 将供电模块从面包板转移到洞洞板或 PCB，提高可靠性；
- 完善多级菜单，实现时间设置、蓝牙配置和步数清零；
- 优化计步算法，提高不同佩戴姿态下的准确率；
- 增加低功耗模式，提高电池续航；
- 开发手机端上位机，实现数据可视化；
- 增加 RTC 模块，实现断电后时间保持；
- 将系统封装成可佩戴外壳，提高实物完整度。

---

## 十四、结论

本项目的创新点并不体现在单一器件的复杂程度上，而是体现在多个常用嵌入式模块的系统集成与协同设计上。通过 STM32F103C8T6 主控、OLED 显示、MPU6050 姿态采集、HC-05 蓝牙通信、EC11 编码器交互、FreeRTOS 多任务调度和面包板供电验证，项目形成了一个功能完整、成本较低、可扩展性较强的智能手表原型。

该设计能够较好地体现嵌入式课程设计中“硬件连接、底层驱动、实时调度、人机交互和系统调试”的综合能力，具有较好的教学实践价值和后续拓展空间。
