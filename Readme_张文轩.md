# STM32 智能手表项目创新点简要报告

## 一、项目名称

基于 STM32F103C8T6 的智能手表设计

## 二、创新点概述

本项目以 STM32F103C8T6 为主控，结合 OLED、MPU6050、HC-05 蓝牙、EC11 编码器和锂电池供电模块，完成了一个低成本、可扩展、可演示的智能手表原型。项目创新主要体现在模块化开发、供电验证、实时任务调度、人机交互和运动数据处理等方面。

## 三、主要创新点

### 1. 渐进式开发方式

项目按照 OLED 显示、MPU6050 传感器、HC-05 蓝牙、FreeRTOS 多任务、计步与菜单功能的顺序逐步开发。每完成一个模块后再加入下一个模块，降低了调试难度，也方便定位问题。

### 2. 面包板供电验证方案

供电模块采用面包板连接，使用 TP4056 进行锂电池充电保护，使用 HT7333 将电池电压稳定为 3.3V。该方式便于前期调试和线路修改，在接入 STM32 前可以先用万用表确认电压是否正常，提高了硬件安全性。

### 3. OLED 与 MPU6050 共用 I2C 总线

OLED 和 MPU6050 共同使用 STM32 的 I2C1 总线，SCL 接 PB6，SDA 接 PB7。两个模块通过不同地址区分，节省了 STM32 引脚资源，也简化了硬件连接。

### 4. FreeRTOS 多任务设计

系统引入 FreeRTOS，将时间更新、传感器读取、OLED 显示、菜单处理和蓝牙通信划分为不同任务。相比普通裸机循环，多任务设计提高了系统实时性和扩展性，也减少了各功能之间的阻塞。

### 5. EC11 编码器菜单交互

项目使用 EC11 旋转编码器实现菜单控制。旋转编码器可以用于页面切换、菜单选择和确认操作，使智能手表具备基本的人机交互能力，提升了演示效果。

### 6. 蓝牙数据通信

通过 HC-05 蓝牙模块，STM32 可以将传感器数据发送到手机端。用户可通过蓝牙串口助手查看加速度、姿态角、温度和计步数据，实现了本地显示与手机端显示的结合。

### 7. 基于 MPU6050 的计步功能

项目利用 MPU6050 的三轴加速度数据进行步伐检测，通过加速度变化判断运动状态，实现了基础计步功能。该设计没有增加额外传感器，降低了成本。

### 8. 软硬件协同调试

硬件部分完成供电、接线和模块测试，软件部分完成外设驱动、任务调度和功能逻辑。通过统一引脚表和分阶段测试流程，减少了软硬件接口不一致导致的问题。

### 9. 低成本可扩展设计

项目使用常见模块搭建系统，成本低、获取方便，适合作为课程设计和嵌入式综合实验平台。后续可以继续扩展 RTC、低功耗模式、手机上位机和 PCB 设计。

## 四、创新点总结

本项目的创新点主要体现在：

- 采用分阶段开发，便于调试和验收；
- 使用面包板完成电源模块验证，提高安全性；
- 通过 I2C 总线共享节省硬件资源；
- 使用 FreeRTOS 实现多任务协同；
- 使用 EC11 编码器实现菜单交互；
- 使用 HC-05 实现手机端数据查看；
- 利用 MPU6050 实现基础计步功能；
- 采用低成本模块实现完整智能手表原型。

## 五、结论

本项目并不是单纯完成几个模块的简单连接，而是将显示、传感器、通信、交互、供电和实时任务调度整合到一个完整系统中。项目具有较好的实物展示效果，也体现了嵌入式系统课程设计中软硬件协同开发的特点。
