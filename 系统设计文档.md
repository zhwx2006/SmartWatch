**2 系统方案设计**

**2.1 系统总体架构**

本系统以 STM32F103C8T6 最小系统板为主控，采用 I2C 总线连接 0.96 寸 OLED 显示屏和 MPU6050 姿态传感器，通过 EC11 旋转编码器完成人机交互，通过蓝牙串口模块实现数据同步。电源部分采用 3.7V 锂电池供电，经 TP4056 充电保护模块和 HT7333-3.3V LDO 稳压后输出 3.3V，为主控与外围模块供电。

供电链路：
Type-C 5V → TP4056 充电保护 → 3.7V 锂电池 → 电源开关 → HT7333-3.3V → 3.3V 系统电源

主控外设拓扑：

```
STM32F103C8T6
├─ I2C 总线 PB6/PB7 → OLED 显示屏、MPU6050
├─ PA0/PA1/PA2 → EC11 旋转编码器
├─ PA9/PA10 USART1 → HC-05 蓝牙模块
└─ PA3 → 电池电压 ADC 检测（可选）
```

**2.2 硬件框图**

```
Type-C 5V充电输入
↓
TP4056充电保护 ←→ 3.7V锂电池
↓
电源开关S1
↓
HT7333 LDO输出3.3V
│
└──── STM32F103C8T6主控
     │
     ┌───────┴───────┬───────────┬───────────┐
I2C(PB6/PB7)    PA0/PA1/PA2   USART1(PA9/10)   PA3分压ADC
     │             │             │               │
┌──────┴──────┐  EC11编码器   HC-05蓝牙    电池电压检测
OLED显示屏  MPU6050传感器
```

**2.3 软件流程图**

1.  上电初始化流程：外设初始化（I2C/USART/ADC/EXTI）→ FreeRTOS 初始化 → 创建 4 个业务任务 + I2C 互斥锁 → 启动调度器，多任务并行运行

2.  Task_KeyEncoder 流程：10ms 轮询编码器中断状态 → 判断正 / 反转切换页面；计时识别短按确认、长按 1s 返回主页

3.  Task_Sensor 流程：200ms 定时读取 MPU6050 三轴数据→滤波处理→计步判定；ADC 读取分压值换算电池电压、软件 RTC 计时累加

4.  Task_Display 流程：100ms 读取页面索引、步数、时间、电压、传感器数据 → 调用 OLED 驱动刷新对应菜单界面

5.  Task_Bluetooth 流程：50ms 轮询串口接收缓存 → 解析 GET/RESETSTEP/SETTIME 指令；定时打包数据上传至手机蓝牙串口

**3 详细设计文档**

**3.1 硬件部分**

**3.1.1 电路原理图**

**（1）电源管理电路**

TP4056 模块：
- B+ → 锂电池正极
- B- → 锂电池负极
- OUT+ → 电源开关 S1 → HT7333 VIN
- OUT- → 系统 GND

HT7333-3.3V：
- VIN → 电源开关输出
- GND → 系统 GND
- VOUT → 3.3V 系统电源

滤波电容配置：
- VIN-GND：10uF + 0.1uF
- VOUT-GND：10uF + 0.1uF

说明：调试阶段可先用 USB 给 STM32 最小系统板供电，待功能稳定后再接入锂电池、TP4056 和 HT7333。

**（2）I2C 总线电路**

```
STM32 PB6 / I2C1_SCL ───┬── OLED SCL
                         └── MPU6050 SCL
STM32 PB7 / I2C1_SDA ───┬── OLED SDA
                         └── MPU6050 SDA
3.3V ───────────────────┬── OLED VCC
                         └── MPU6050 VCC
GND ───────────────────┬── OLED GND
                         └── MPU6050 GND
```

可选上拉：SCL、SDA 各接 4.7kΩ 到 3.3V。若模块自带上拉电阻，可不额外添加。

**（3）编码器、蓝牙与 ADC 检测电路**

| 功能 | STM32 引脚 | 外设引脚 | 备注 |
|------|-----------|---------|------|
| 编码器 A 相 | PA0 | CLK/A | 用于判断旋转脉冲 |
| 编码器 B 相 | PA1 | DT/B | 用于判断旋转方向 |
| 编码器按键 | PA2 | SW | 输入上拉，软件消抖 |
| 蓝牙发送 | PA9 / USART1_TX | 蓝牙 RXD | 建议串 1kΩ 保护电阻 |
| 蓝牙接收 | PA10 / USART1_RX | 蓝牙 TXD | TX/RX 交叉连接 |
| 电池检测 | PA3 / ADC | 电阻分压中点 | R1=100kΩ，R2=100kΩ |

**3.1.2 关键元件参数计算**

**（1）系统电流估算**

| 模块 | 估计电流 | 说明 |
|------|---------|------|
| STM32F103C8T6 | 20--40mA | 主控运行电流 |
| OLED | 10--30mA | 与显示内容和亮度有关 |
| MPU6050 | 3--5mA | 姿态检测 |
| EC11 编码器 | <1mA | 输入器件 |
| 蓝牙模块 | 30--50mA | 连接时电流较大 |
| 合计 | 约 80--130mA | 按最大约 130mA 设计 |

系统最大工作电流按 130mA 估算，HT7333 主要为 STM32、OLED、MPU6050 和编码器供电；若蓝牙工作时出现复位，可改为 USB 供电或让蓝牙单独供电。

**（2）HT7333 功耗计算**

LDO 功耗公式：P = (Vin - Vout) × Iout

满电锂电池 Vin = 4.2V，Vout = 3.3V，Iout = 0.13A
P = (4.2 - 3.3) × 0.13 = 0.117W

0.117W 功耗较低，满足课程设计短时间演示需求。若长时间运行或蓝牙电流偏大，应增加输出电容或更换更大电流稳压器。

**（3）I2C 上拉电阻计算**

取 R = 4.7kΩ，VCC = 3.3V
I = V / R = 3.3 / 4700 ≈ 0.70mA

4.7kΩ 上拉电阻电流适中，能够兼顾功耗和总线稳定性，适合本项目短距离 I2C 连接。

**（4）电池电压检测分压计算**

R1 = 100kΩ，R2 = 100kΩ
Vadc = Vbat × R2 / (R1 + R2) = Vbat / 2
锂电池满电 Vbat = 4.2V，则 Vadc = 2.1V < 3.3V，ADC 输入安全。
STM32 反算公式：Vbat = ADC_Value / 4095 × 3.3 × 2

**3.1.3 PCB 布局规划**

本项目可采用洞洞板焊接固定，也可使用立创 EDA 绘制简单模块转接 PCB。PCB 不追求复杂布线，重点保证电源稳定、I2C 短线、模块布局清晰。

推荐尺寸：70mm × 45mm 左右

```
┌──────────────────────────────┐
│        OLED 显示区           │
├──────────────────────────────┤
│       STM32 主控区           │
├───────────────┬──────────────┤
│ MPU6050传感器区 │  EC11 输入区 │
├───────────────┴──────────────┤
│ TP4056+HT7333电源区  蓝牙区  │
└──────────────────────────────┘
```

| 布局 / 布线项目 | 设计要求 |
|----------------|---------|
| OLED | 放在正面上方，便于显示和验收观察 |
| STM32 | 放在中央，缩短到各模块的连线 |
| MPU6050 | 靠近 STM32，远离蓝牙和电源开关，I2C 线尽量短 |
| EC11 编码器 | 放在板边，方便手指旋转操作 |
| 蓝牙模块 | 放在边缘，天线附近避免大面积铺铜和金属遮挡 |
| 电源模块 | TP4056、HT7333、电池接口、开关集中放在下方 |
| 电源线 | 3.3V 和 GND 线加粗，建议 0.8--1.2mm |
| 信号线 | I2C、UART、编码器信号线可用 0.25--0.3mm |
| 地线 | PCB 建议底层铺 GND；洞洞板建议焊一条粗 GND 母线 |

**3.1.5 硬件设计结论**

本硬件方案采用模块化设计，以 STM32F103C8T6 最小系统板为核心，通过 I2C 总线连接 OLED 和 MPU6050，通过 EC11 编码器实现菜单输入，通过蓝牙模块实现数据同步。电源部分使用 TP4056 完成锂电池充电保护，使用 HT7333-3.3V 输出稳定 3.3V。PCB 或洞洞板布局按显示区、主控区、传感器区、交互区和电源区分区设计，能够满足课程设计提交与实物演示需求。

**（2）分层软件架构**

软件采用分层架构，从上至下分为三层：

1. 应用层（FreeRTOS 多任务）
   - Task_DisplayOLED：界面刷新、菜单页面切换逻辑
   - Task_SensorMPU6050：数据采集、计步算法、电池电压 ADC 采集
   - Task_Bluetooth：串口数据接收、蓝牙命令解析与回复
   - Task_KeyEncoder：EC11 编码器旋转与按键状态机处理（短按 / 长按）

2. 驱动层（BSP）
   - oled.c/mpu6050.c：底层 I2C 读写、传感器初始化、数据寄存器操作
   - gpio.c/adc.c：外部中断触发、ADC 软件触发单次转换

3. 硬件抽象层
   - HAL 库：STM32F103 标准 HAL 外设驱动（I2C, UART, ADC, EXTI）

**3.2.2 外设配置方案**

| 功能 | 通信接口 / 引脚 | 关键参数配置 |
|------|----------------|------------|
| OLED 显示 | I2C1 (PB6-SCL, PB7-SDA) | 地址 0x3C（7 位），400kHz 高速模式 |
| MPU6050 | I2C1 (与 OLED 共用总线) | 地址 0x68，关闭睡眠模式，量程 ±2g |

**3.2.3 任务调度设计（FreeRTOS 任务划分）**

系统共划分 4 个用户任务，并创建了 1 个互斥锁（I2C_Mutex）用于保护 I2C 总线（防止 OLED 与 MPU6050 同时操作导致数据错乱）。

| 任务名称 | 核心功能 | 优先级 | 栈大小 (Words) | 执行周期 |
|---------|---------|--------|---------------|---------|
| Task_KeyEncoder | 检测旋转方向、按键短按 / 长按（长按 1 秒回主页） | High | 128 | 10ms 轮询 |
| Task_Sensor | 读取 MPU6050 加速度 / 陀螺仪、简易计步、读取电池电压、软件模拟 RTC 时间累加 | Normal | 128 | 200ms |
| Task_Display | 根据当前页面索引刷新 OLED（主页 / 传感器页 / 运动页） | Low | 128 | 100ms |
| Task_Bluetooth | 解析手机 APP 下发指令（GET/RESETSTEP/SETTIME），上传数据包 | Normal | 256 | 50ms |

**3.2.4 关键算法描述**

1. EC11 编码器驱动逻辑（中断法）

   配置 PA0 和 PA1 为下降沿触发中断。在中断服务回调中，若当前触发引脚为 PA0，则立即读取 PA1 电平。若 PA1 为高电平判定为正转（顺时针），反之为反转（逆时针）。按键 PA2 短按触发 "确认 / 翻页"，长按（>1000ms）触发 "返回主页"。

2. 简易计步算法

   基于 MPU6050 的 Z 轴加速度数据。设定动态阈值 500（经简单滤波处理），当 Z 轴加速度由正值穿越到负值（或由负穿越到正）且超过阈值时，步数寄存器累加 1。此算法虽简陋但适合课程演示。

3. 蓝牙通信协议

   下发指令（手机→手表）：
   - GET：手表回传当前时间、步数、三轴加速度数据。
   - RESETSTEP：步数清零。
   - SETTIME,12,30,00：校准时间。

   上传数据（手机→手机）：格式为 TIME=12:30,STEP=1234,AX=120,AY=-50,AZ=16300
